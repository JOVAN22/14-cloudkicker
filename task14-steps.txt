 
Task 14 – Deploy Flask Application on GCP VM with Nginx
 
Cloud Provider : Google Cloud Platform (GCP)
OS             : Ubuntu 22.04 LTS
External IP    : 136.115.182.88
Confirmation   : Flask bound to 127.0.0.1:5000, Nginx on 80
 


 
STEP 1 — CREATE A FREE VM IN GOOGLE CLOUD
 

Open the Google Cloud Console:
https://console.cloud.google.com

Navigate to:
Compute Engine -> VM Instances -> Create Instance

VM Configuration:
-----------------
  Name         : cloudkicker-vm
  Region       : us-central1 (Free tier eligible)
  Machine Type : e2-micro (2 vCPU, 1 GB RAM)
  Boot Disk    : Ubuntu 22.04 LTS
  Disk Size    : 20 GB
  Firewall     : Allow HTTP traffic (port 80)

After the VM is created, note the External IP address.
External IP used: 136.115.182.88


 
STEP 2 — SSH FROM KALI LINUX INTO THE VM
 

-- Generate SSH key in Kali --

  $ ssh-keygen -t rsa -b 4096
  (Press Enter for all prompts)

  Keys created:
    ~/.ssh/id_rsa        (private key)
    ~/.ssh/id_rsa.pub    (public key)

-- Display the public key --

  $ cat ~/.ssh/id_rsa.pub
  Copy the entire output line.

-- Add the key to GCP VM --

  Google Cloud Console
  -> Compute Engine -> VM Instances
  -> Click cloudkicker-vm -> Edit
  -> Scroll to SSH Keys -> Add Item
  -> Paste the public key -> Save

-- Connect from Kali --

  $ ssh Jovan@136.115.182.88

  Successful login shows:
  Jovan@cloudkicker-vm:~$




 
STEP 3 — INSTALL PREREQUISITES ON THE VM
 

  $ sudo apt update
  $ sudo apt upgrade -y
  $ sudo apt install git python3 python3-venv python3-pip nginx -y

-- Verify Nginx is running --

  $ sudo systemctl status nginx
  Expected: Active: active (running)


 
STEP 4 — CLONE THE FLASK REPOSITORY
 

  $ git clone https://github.com/jovanjacob58/12-fsd-flask.git
  $ cd 12-fsd-flask/12-fsd-flask


 
STEP 5 — CREATE VIRTUAL ENVIRONMENT + INSTALL DEPENDENCIES
 

-- Create virtual environment --

  $ python3 -m venv venv

-- Activate virtual environment --

  $ source ~/12-fsd-flask/12-fsd-flask/venv/bin/activate

  Prompt changes to:
  (venv) Jovan@cloudkicker-vm:~/12-fsd-flask/12-fsd-flask$

-- Install dependencies --

  $ pip install -r requirements.txt


 
STEP 6 — RUN FLASK ON LOCALHOST ONLY (NOT PUBLIC)
 

  $ nohup flask run --host=127.0.0.1 --port=5000 > ~/flask.log 2>&1 &

  Flask is bound to 127.0.0.1:5000 only.
  It is NOT accessible from the internet directly.
  Only Nginx can reach it internally on the VM.

-- Verify Flask is running --

  $ curl http://127.0.0.1:5000
  Expected: Redirecting to /login (Flask is working correctly)


 
STEP 7 — CONFIGURE NGINX AS REVERSE PROXY
 

-- Edit the default Nginx configuration --

  $ sudo nano /etc/nginx/sites-available/default

  Delete everything and replace with:

  -------------------------------------------------------
  server {
      listen 80 default_server;
      listen [::]:80 default_server;
      server_name _;

      location / {
          proxy_pass http://127.0.0.1:5000;
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header X-Forwarded-Proto $scheme;
          proxy_redirect off;
      }
  }
  -------------------------------------------------------

  Save: Ctrl+O -> Enter -> Ctrl+X

-- Test and reload Nginx --

  $ sudo nginx -t
  Expected: syntax is ok / test is successful

  $ sudo systemctl reload nginx


 
STEP 8 — OPEN FIREWALL FOR PORT 80 (GCP Console)
 

  GCP Console -> VPC Network -> Firewall
  Verify rule exists allowing TCP port 80 from 0.0.0.0/0

  Note: Port 5000 is NOT opened in firewall.
  Flask only listens on 127.0.0.1 so it is already protected.


 
STEP 9 — VERIFY PORTS
 

  $ ss -tlnp | grep -E '5000|80'

  Expected output:
    LISTEN  0.0.0.0:80      -> Nginx   (public-facing)
    LISTEN  127.0.0.1:5000  -> Flask   (localhost only)

  Confirmation: Flask bound to 127.0.0.1:5000, Nginx on 80

  Nginx proxies the request internally to Flask on port 5000.
